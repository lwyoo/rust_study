# Rust 프로그래밍 언어 개요

---

## 1️⃣ Rust란?

**Rust는 메모리 안전성과 성능을 모두 갖춘 시스템 프로그래밍 언어입니다.**

- **등장 시기**: 2010년 발표, 2015년 1.0 공식 릴리스
- **개발 목표**: C/C++의 성능 + 메모리 안전성
- **주요 사용 분야**: 
  - 시스템 프로그래밍 (OS, 커널)
  - 내장 시스템 (임베디드)
  - 웹 서버 (Actix, Tokio)
  - 블록체인 (Solana, Polkadot)
  - 자동차 소프트웨어 (당신이 일하는 분야! 🚗)

## 2️⃣ 메모리 안전성 - Rust vs C/C++

### C/C++의 문제점 ❌

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr = (int *)malloc(sizeof(int));
    *ptr = 10;
    
    free(ptr);
    free(ptr);  // ❌ 이중 해제 (Double Free)
    
    printf("%d", *ptr);  // ❌ Use-After-Free (메모리 접근)
    
    return 0;
}
```

**발생할 수 있는 문제:**
- **Memory Leak**: 해제하지 않은 메모리가 계속 점유
- **Buffer Overflow**: 배열 범위를 넘어 접근
- **Segmentation Fault**: 할당되지 않은 메모리 접근
- **Use-After-Free**: 해제된 메모리 접근
- **Data Race**: 멀티스레드에서 동시 접근

### Rust의 해결책 ✅

```rust
fn main() {
    let ptr = vec![10];  // 메모리 자동 할당
    println!("{}", ptr[0]);  // 10
    
    // ptr이 스코프를 벗어나면 자동으로 메모리 해제
}  // ✅ 자동 해제됨
```

**Rust의 장점:**
- **컴파일 단계에서 모든 메모리 문제 감지**
- **런타임 오류 방지**
- **메모리 해제 자동화**
- **안전한 멀티스레드 프로그래밍**

## 3️⃣ Rust의 핵심: 오너십(Ownership) 시스템

### 규칙 1️⃣: 각 값은 하나의 오너만 가짐

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // s1의 소유권이 s2로 이동 (Move)
    
    // println!("{}", s1);  // ❌ 에러! s1은 더 이상 소유권 없음
    println!("{}", s2);    // ✅ 가능
}
```

### 규칙 2️⃣: 스코프를 벗어나면 자동으로 메모리 해제

```rust
fn main() {
    {
        let s = String::from("hello");
        println!("{}", s);  // hello
    }  // ← 여기서 자동으로 메모리 해제됨
    
    // println!("{}", s);  // ❌ 에러! s는 스코프 밖
}
```

### 규칙 3️⃣: 참조(&)를 사용하여 소유권 유지

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = &s1;  // s1의 참조만 받음 (소유권 이동 아님)
    
    println!("{}", s1);  // ✅ 가능 (s1이 여전히 소유자)
    println!("{}", s2);  // ✅ 가능 (참조)
}
```

## 4️⃣ C/C++ vs Rust 비교표

| 항목 | C/C++ | Rust |
|------|-------|------|
| **메모리 관리** | 수동 (malloc/free) | 자동 (오너십) |
| **컴파일 체크** | 거의 없음 | 매우 엄격함 |
| **런타임 에러** | 빈번 (Segfault 등) | 거의 없음 |
| **배우기 어려움** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **성능** | ⭐⭐⭐⭐⭐ 매우 빠름 | ⭐⭐⭐⭐⭐ 매우 빠름 |
| **보안** | ⭐⭐ 위험함 | ⭐⭐⭐⭐⭐ 안전함 |
| **개발 속도** | ⭐⭐⭐⭐ 빠름 | ⭐⭐⭐ 느림 |

## 5️⃣ Rust의 장점 ✅

### 1. 메모리 안전성 보장

```rust
// 컴파일 타임에 모든 메모리 문제 감지
fn safe_function() -> String {
    let s = String::from("hello");
    s  // ✅ 자동으로 해제됨
}
```

### 2. 성능 우수

```rust
// C/C++과 동일한 수준의 성능
// 런타임 오버헤드 거의 없음
let arr = vec![1, 2, 3, 4, 5];
```

### 3. 안전한 동시성

```rust
use std::thread;

fn main() {
    let data = vec![1, 2, 3];
    
    let handle = thread::spawn(|| {
        // ✅ 컴파일러가 자동으로 데이터 안전성 보장
        println!("{:?}", data);
    });
    
    handle.join().unwrap();
}
```

### 4. 현대적 기능

- 패턴 매칭
- 제너릭
- 트레이트 (Trait)
- 매크로
- 모듈 시스템

## 6️⃣ Rust의 단점 ❌

### 1. 높은 진입장벽

```rust
// 학습 곡선이 가파름
fn main() {
    let x = 5;           // 변수 (immutable)
    let mut y = 5;       // 가변 변수
    let z: i32 = 5;      // 타입 지정
    let w = 5_i32;       // 다른 타입 표기법
    // 이런 것들이 많음...
}
```

### 2. 엄격한 컴파일러

```rust
fn main() {
    let s = String::from("hello");
    let s2 = s;
    
    println!("{}", s);  // ❌ 컴파일 에러!
    // error[E0382]: borrow of moved value: `s`
}
```

### 3. 개발 속도 느림

```rust
// C/C++: 몇 줄
// Rust: 생각해야 할 것이 많음 (소유권, 빌로우, 라이프타임 등)
```

### 4. 생태계 작음

- C/C++: 50년 이상의 라이브러리
- Python: 매우 풍부한 라이브러리
- Rust: 아직 성장 중

## 7️⃣ Rust가 해결하는 실제 문제들

### 예제 1: Buffer Overflow 방지

```rust
// C의 위험한 코드
char buffer[10];
strcpy(buffer, "hello world");  // ❌ 버퍼 오버플로우!

// Rust: 컴파일 단계에서 감지
let s = String::from("hello world");
// ✅ 자동으로 메모리 할당
```

### 예제 2: Null Pointer Dereference 방지

```rust
// C: NULL 포인터로 인한 크래시
int *ptr = NULL;
printf("%d", *ptr);  // ❌ Segmentation Fault

// Rust: Option 타입으로 안전하게 처리
let value: Option<i32> = None;
match value {
    Some(x) => println!("{}", x),
    None => println!("값이 없음"),  // ✅ 안전한 처리
}
```

### 예제 3: Data Race 방지

```rust
use std::sync::Arc;
use std::sync::Mutex;
use std::thread;

fn main() {
    let counter = Arc::new(Mutex::new(0));
    let mut handles = vec![];
    
    for _ in 0..10 {
        let counter = Arc::clone(&counter);
        let handle = thread::spawn(move || {
            let mut num = counter.lock().unwrap();
            *num += 1;
        });
        handles.push(handle);
    }
    
    for handle in handles {
        handle.join().unwrap();
    }
    
    println!("결과: {}", *counter.lock().unwrap());  // ✅ 안전한 병렬 처리
}
```
