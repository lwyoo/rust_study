🦀 Rust Core Concepts

5-minute cheat sheet for quick review

Ownership

각 값은 하나의 owner를 가짐
Owner가 스코프를 벗어나면 자동 drop
소유권은 move 또는 clone
s1
→ move →
s2 ✓
let s1 = String::from("hi"); let s2 = s1; // move // println!("{}", s1); ❌ error

Borrowing

불변 참조: &T (여러 개 가능)
가변 참조: &mut T (하나만)
두 타입을 동시에 사용 불가
여러 &
또는
하나의 &mut
let mut s = String::from("hi"); let r1 = &s; // ok let r2 = &s; // ok let r3 = &mut s; // ❌ error

Pattern Matching

match로 모든 경우 처리
Enum 변수 분해 가능
if let으로 간략하게
match msg { Message::Quit => "quit", Message::Text(s) => s, _ => "other", } // if let Message::Text(s) = msg { }

Error Handling

성공/실패: Result<T, E>
값 있음/없음: Option<T>
? operator로 에러 전파
fn parse(s: &str) -> Result<i32, E> { s.parse()? // ? = early return } match file.read() { Ok(data) => use(data), Err(e) => handle(e), }

Traits & Generics

Trait = 인터페이스 (메서드 서명)
Generic = 타입 변수 T
Trait bounds로 제약
trait Animal { fn speak(&self); } fn print<T: Animal>(a: &T) { a.speak(); } impl Animal for Dog { ... }

Lifetime

참조의 유효 범위
컴파일러가 자동 추론 (대부분)
명시 필요시: 'a 사용
보통 함수에서 자동으로 결정됨
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str { if x.len() > y.len() { x } else { y } }

Copy vs Clone

Copy: 기본 타입 (i32, bool)
→ Assignment 후에도 두 값 모두 유효
Clone: String, Vec (명시적)
→ 깊은 복사, s.clone()
let x = 5; let y = x; // Copy ✓ println!("{}", x); let s = String::from("hi"); let s2 = s.clone(); // Clone ✓

Collections

Vec<T>: 동적 배열
HashMap<K,V>: 해시맵
Iterator: .iter(), .map(), .filter()
let v = vec![1,2,3]; let sum: i32 = v.iter() .filter(|&&x| x > 1) .map(|x| x * 2) .sum(); // 10
